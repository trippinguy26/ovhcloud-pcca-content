# Domain 09 — Operations

> **Domain code**: `OPS`
> **Total LOs**: 11 (4 K + 5 S + 2 A)
> **Calibration**: Day-2 operational essentials: native monitoring, cost tracking, quotas, support, and status page. Advanced observability (Log Data Platform deep usage, custom metrics, third-party APM integration) is named for awareness only and belongs to Professional. FinOps optimization strategies are explicitly out of scope.

## Knowledge (savoir)

| Code | Learning Outcome |
|---|---|
| `LO-OPS-K01` | Identify the native observability surfaces of OVHcloud Public Cloud (Manager dashboards, instance metrics, project consumption views). |
| `LO-OPS-K02` | Identify the OVHcloud Log Data Platform and its positioning for centralized log collection. |
| `LO-OPS-K03` | Define quotas and limits in a Public Cloud project and explain how to request increases. |
| `LO-OPS-K04` | Identify the OVHcloud support channels and SLA structure available depending on the customer contract. |

## Skills (savoir-faire)

| Code | Learning Outcome |
|---|---|
| `LO-OPS-S01` | Read and interpret instance-level metrics (CPU, RAM, network, disk) from the Manager. |
| `LO-OPS-S02` | Track project consumption over a billing period and identify the top cost contributors. |
| `LO-OPS-S03` | Check current quotas on a project and request a quota increase through the appropriate channel. |
| `LO-OPS-S04` | Consult the OVHcloud status page and interpret an ongoing incident notification. |
| `LO-OPS-S05` | Open a support ticket with adequate diagnostic information (project ID, instance ID, timestamps, logs excerpt). |

## Attitudes (savoir-être / posture)

| Code | Learning Outcome |
|---|---|
| `LO-OPS-A01` | Apply a cost-review reflex at regular intervals (weekly or monthly) on any production project. |
| `LO-OPS-A02` | Recommend a baseline observability posture for any new project (key metrics watched, alert thresholds, log retention). |

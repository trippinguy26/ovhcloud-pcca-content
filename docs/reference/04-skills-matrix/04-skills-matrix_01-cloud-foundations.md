# Domain 01 — Cloud Foundations

> **Domain code**: `FND`
> **Total LOs**: 9 (7 K + 0 S + 2 A)
> **Calibration**: Conceptual framing module. Defines vocabulary, models, and OVHcloud positioning. Establishes the strategic narrative that the rest of the curriculum builds upon.

## Knowledge (savoir)

| Code | Learning Outcome |
|---|---|
| `LO-FND-K01` | Define cloud computing and the three service models (IaaS, PaaS, SaaS) with concrete OVHcloud examples for each. |
| `LO-FND-K02` | Distinguish the four cloud deployment models (public, private, hybrid, multi-cloud) and identify their typical use cases. |
| `LO-FND-K03` | Explain the OpEx vs CapEx shift induced by cloud consumption and its impact on IT budgeting. |
| `LO-FND-K04` | Describe the shared responsibility model and identify what the customer and the cloud provider are each responsible for. |
| `LO-FND-K05` | Identify the strategic positioning of OVHcloud against US hyperscalers (sovereignty, pricing transparency, open standards, no egress fees). |
| `LO-FND-K06` | Explain at a high level what OpenStack is, why OVHcloud Public Cloud is built on it, and what this means for the customer (portability, open APIs, ecosystem). |
| `LO-FND-K07` | List the main service families of the OVHcloud Public Cloud catalog (Compute, Storage, Network, Databases, Containers, AI, Analytics, Security & Identity, Operations, DevOps tooling). |

## Skills (savoir-faire)

*No skill outcomes for this domain.*

Cloud Foundations is purely conceptual and acts as the framing module. Hands-on activity starts in Domain 02 (PCI Foundation).

## Attitudes (savoir-être / posture)

| Code | Learning Outcome |
|---|---|
| `LO-FND-A01` | Justify when Public Cloud is the appropriate target for a given workload, and when it is not. |
| `LO-FND-A02` | Position OVHcloud Public Cloud credibly in a conversation with a stakeholder familiar with AWS or Azure, without defensiveness. |

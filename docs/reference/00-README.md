## How to use this reference

- **Quick decision check**: read 01 (Executive Summary) and 03 (Scope Statement) only.
- **Production handoff**: in Phase 2, each Learning Outcome (`LO-XXX-YYY`) in the matrix will be traced to specific training content (slide, demo, lab, exam question).

## Naming and identification conventions

- **Program** (parent): *Public Cloud Core* — spans Associate, Professional, Specialist levels.
- **Certification** (commercial, public-facing): *OVHcloud - Public Cloud - Core Associate*.
- **"Core" defined**: the IaaS layer of OVHcloud Public Cloud — OpenStack-native services plus adjacent essentials and native operation tooling (see Scope Statement for the operational definition).
- **Learning Outcome ID format**: `LO-[DOMAIN_3_LETTERS]-[CATEGORY][NUMBER]`
  - Categories: `K` (Knowledge), `S` (Skill), `A` (Attitude / posture)
  - Example: `LO-CMP-S03` = Compute domain, Skill outcome #3
- **Module ID format**: `[DAY].[MODULE_INDEX]` (e.g. `1.2` for day 1, second module)
